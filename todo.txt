The following document describes work on the [Eidos](https://github.com/NateWr/Eidos/) theme that remains to be discussed, prioritized, and completed.

This list is compiled against the draft prepared for review on 19 June 2026. To test the draft, you must have:

- [Eidos](https://github.com/NateWr/Eidos/) installed, `main` branch, commit `a2ffe00`.
- Run the install and build steps for `Eidos` (see readme).
- The `NateWr/eidos` fork of `ojs`, `pkp-lib`, and `ui-library`. Recompile the frontend UI components for OJS (`npm run build:frontend`).
- If you want test data, you can use the files in [./dataset](./dataset/). These assume you are running a local install at `localhost:8000`.
- The main journal with test data is Journal of Public Knowledge.

Design revisions/discussions:
=====
- When an article appears in a list (issue table of contents, search, latest
  articles), show the galleys in the same Downloads dropdown used on the
  article page.
- In the issue table of contents, when an issue has a cover but no description,
  there is a large empty space before the articles. To fix this, I have to adjust
  the HTML structure of the articles. I think the best solution is to never show
  article cover images in the table of contents. What do people think about not
  showing cover images in issue table of contents?
- Explore design options to visually separate metadata in the article sidebar?

  I don't need to do this? If not, just use pagination.blade.
- When is a good time to align pagination methods across the frontend, so
  we can use one simple template, instead of all the different URL patterns?
- Should the CustomBlockPlugin make its blocks available as Homepage Blocks?
